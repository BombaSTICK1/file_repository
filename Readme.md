1. Бизнес-цель

Позволить пользователям хранить, организовывать и отслеживать историю изменений файлов в иерархической (древовидной) структуре, подобной файловой системе, с возможностью просмотра и восстановления предыдущих версий.

Ключевые бизнес-требования:

Сократить риск потери данных при случайном изменении/удалении.
Обеспечить прозрачную историю изменений файлов.
Упростить навигацию по большим коллекциям документов через папки.

2. Функциональные требования

ID                          Требование
FR-01   Пользователь может зарегистрироваться и войти в систему.

FR-02   Пользователь может создавать, переименовывать и удалять папки.

FR-03   Пользователь может загружать файлы в папку.

FR-04   При каждом обновлении файла (повторная загрузка с тем же именем) система сохраняет новую версию, не удаляя предыдущие.

FR-05   Пользователь может просмотреть список версий файла и выбрать любую для просмотра или скачивания.

FR-06   Пользователь может перемещать файлы и папки внутри дерева.

FR-07   Система отображает древовидную структуру папок и файлов (как в проводнике).

3. Нефункциональные требования

Категория                   Требование
Производительность  Ответ API должен возвращаться за ≤ 500 мс при количестве файлов ≤ 1000.

Безопасность        Доступ к файлам только у владельца. Авторизация через JWT.

Надёжность          Версии файлов не должны теряться при сбое системы.

Интерфейс           Веб-интерфейс на React, адаптивный (работает на десктопе и планшете).

Совместимость       Поддержка файлов до 50 МБ (на старте).

4. REST API (основные эндпоинты)

Метод       Endpoint        Описание

POST    /auth/register  Регистрация пользователя

POST    /auth/login     Получение JWT-токена

GET     /folders/tree   Получить всё дерево папок и файлов

POST    /folders        Создать папку ({ name, parentId })

POST    /files/upload   Загрузить файл (multipart/form-data, указать folderId)

GET     /files/{fileId} Получить список версий файла
        /versions    

GET     /files/{fileId} Скачать конкретную версию
        /versions/
        {versionId}

PUT     /items/{itemId} Переместить файл/папку в другую папку
        /move

Все эндпоинты (кроме /auth) защищены токеном.

5. Модель данных (упрощённо)

User
├── id (PK)
├── email
└── password_hash

Folder
├── id (PK)
├── name
├── owner_id → User.id
└── parent_id → Folder.id (NULL = корень)

File
├── id (PK)
├── name
├── folder_id → Folder.id
├── owner_id → User.id
└── current_version_id → FileVersion.id

FileVersion
├── id (PK)
├── file_id → File.id
├── version_number (1, 2, 3...)
├── storage_path (путь в файловой системе или S3)
├── uploaded_at
└── size_bytes
